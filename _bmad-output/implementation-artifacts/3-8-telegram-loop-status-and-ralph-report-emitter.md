# Story 3.8: Telegram Loop Status Messages and Ralph Report Emitter

Status: ready

## Story

As a platform operator,
I want real-time Telegram updates during the Ralph Loop and a structured report when it ends,
so that I can monitor progress passively and act on the output without digging through logs.

## Acceptance Criteria

1. Given the Ralph Loop starts, when initialisation completes, then a Telegram message is sent: workflow name, iteration cap, dashboard URL with one-time token, and inactivity timeout.
2. Given an iteration completes with failures, when the loop posts a status update, then the Telegram message shows: iteration number, which assertions failed (with actual vs expected), runtime errors if any, and "Attempting auto-fix...".
3. Given the loop halts with status `PASS`, then a Telegram message is sent: "✅ Ralph Loop passed — <workflow> ready for approval gate." with link to ralph-report.
4. Given the loop halts with status `EXHAUSTED`, `ABORTED`, or `TIMEOUT`, then a Telegram message is sent with the halt reason and a link to the partial/full ralph-report.
5. Given abort is triggered via Telegram inline button, when the button is tapped, then the abort signal is written to SQLite (Story 3.7), a confirmation message is sent, and the partial report is emitted once the current iteration finishes.
6. Given the loop completes (any halt status), then `ralph-report-<slug>-<timestamp>.md` is written to `_bmad-output/ralph-reports/` and committed to the feature branch.

```markdown
# Ralph Report — <workflow-slug>
Generated: <timestamp>

Iterations run: X / 5

## Requirements Coverage

- Runtime errors: <excerpt>
- Failed assertions: <list>

### Iteration 2
...

## Suggested Next Steps
- <auto-generated based on final failure delta>
```

## Telegram Message Templates

**Loop started:**


**Iteration failed:**
```
🔁 Ralph Loop — Iteration <N>/5
Workflow: <slug>
❌ <assertion failures summary>
⚠️ Runtime: <error excerpt or "none">
🔧 Attempting auto-fix...
```

**Loop passed:**
```
✅ Ralph Loop passed — Iteration <N>/5
Workflow: <slug>
All assertions passed. Ready for approval gate.
📄 Report: <link>
```

**Loop halted:**
```
⛔ Ralph Loop halted — <EXHAUSTED|ABORTED|TIMEOUT>
Workflow: <slug>
Iterations completed: <N>/5
📄 Partial report: <link>
[🔁 Restart loop]
```

## Implementation Summary

- `TelegramNotifier`: wraps Telegram bot API, sends loop status messages with inline buttons
- `AbortButtonHandler`: registers callback for abort inline button, writes to `ralph_loop_signals` table
- `RalphReportEmitter`: assembles `ralph-report.md` from loop record + iteration log + suggested next steps
- `NextStepAdvisor`: LLM call on final failure delta → produces plain-language suggested next steps section of report
- Report written to `_bmad-output/ralph-reports/<slug>-<timestamp>.md`, staged for commit
